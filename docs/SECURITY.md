# Khora Security Policy

## Authorized Use

Use Khora only in:

- systems you own
- environments where you have explicit written authorization
- isolated labs and training setups

## Project Rules

- Keep documentation truthful to the codebase
- Do not document planned features as implemented features
- Keep generated data local unless your engagement rules say otherwise
- Review logs and output directories after local testing

## Local Data

- `logs/` contains runtime log files
- `results/` contains session output
- additional local artifacts may be created by modules depending on environment

## Reporting Security Issues

Include:

1. affected file
2. observed behavior
3. reproduction steps
4. impact
5. suggested fix if available

## Current Review Outcome

During the latest documentation pass, the project docs were updated to clearly separate implemented behavior from planned agent functionality.
